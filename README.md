# Full-Stack Web Development & E-commerce Integrations Archive

Этот репозиторий содержит код (Поиска товаров через Api "apiSearch" и  интеграция эквайринга PayKeeper "apiBank" ), разработанные мной для высоконагруженных e-commerce систем.

Сам код сделан в виде решения которое инегрирванно в шабоны страниц (тк заказчик хотел кастомное оформелние заказа и корзину)

В папках apiBank и apiSearch
находится код котрый использовался в cms WordPress , поэтому сама струтура и файлы будут в виде структуры тем и ядра cms 

## Ключевые проекты 

### 1. Custom E-commerce Engine (Core Logic)
Полноценный цикл оформления заказа, интегрированный в CMS, но реализованный как независимый сервисный слой (~5000 строк кода).
* **Функционал:** Кастомная корзина, многошаговое оформление заказа (Checkout), управление кратностью товара (`folddivision`) и минимальными партиями (`minq`).
* **Особенности:** Полное разделение бизнес-логики и представления. Использование AJAX для бесшовного взаимодействия.

### 2. Высокопроизводительный поиск и Парсинг API
Система реального времени для получения данных о складских остатках от внешних поставщиков (на примере GetChips).
* **Технологии:** Многопоточные запросы через `curl_multi` на PHP, обеспечивающие минимальный Time-to-Response при опросе нескольких API.
* **Математика наценок:** Гибкий алгоритм расчета итоговой цены в зависимости от курса валют ЦБ РФ (автоматический парсинг XML CBR) и типа наценки (процентная/фиксированная, к курсу или к цене).

### 3. Fintech: Интеграция с PayKeeper API
Модуль приема платежей, разработанный в условиях динамически меняющейся документации со стороны банка.


## Технологический стек
* **Backend:** PHP 7.4/8.x (включая многопоточность cURL), MySQL/MariaDB.
* **Frontend:** JavaScript (Native/jQuery для AJAX), CSS3 (адаптивная верстка таблиц).
* **Integrations:** REST API, SOAP/XML, JSON, Banking Gateways.
* **Tools:** Git, работа с инфраструктурой CMS (WordPress/Bitrix) на уровне глубокой кастомизации ядра.

## Чему меня научили эти проекты
1.  **Адаптивность:** Умение быстро переписывать интеграции при внезапном обновлении API сторонних сервисов (кейс с PayKeeper).
2.  **Масштабируемость:** Написание кода, который остается поддерживаемым при объеме более 5000 строк в рамках одного модуля.
3.  **Оптимизация:** Переход от последовательных запросов к параллельным для ускорения работы интерфейса в 5-10 раз.

Код представлен в ознакомительных целях как пример реализации сложной бизнес-логики!
